Abstract

This document records all known errors in the Accessible Rich Internet Applications (WAI-ARIA) 1.1 and the Core Accessibility API Mappings 1.1 specifications.

The errata are numbered, classified as Substantive or Editorial, and listed in reverse chronological order of their date of publication in each category.

Each entry has the following information:

  • A unique entry number
  • The date it was added to the errata page.
  • The section referred to.
  • A description of the problem and correction if applicable.
  • A rationale for making the change (not required for editorial errata).

Substantive corrections are proposed by the Accessible Rich Internet Applications Working Group which has consensus that they are appropriate; they are not to be considered normative unless incorporated into an Edited Recommendation.

To comment on these documents, file issues in the aria GitHub repository for WAI-ARIA, or in the core-aam GitHub repository for Core-AAM; alternatively, send email to public-aria@w3.org. Comments received on W3C Recommendations cannot result in changes to this version of the specification, but may be addressed in these errata or in future versions of WAI-ARIA. The Working Group may not make formal responses to comments but future work undertaken by the Working Group may address comments received on this document. Comments submitted are publicly available in the comment archive.
